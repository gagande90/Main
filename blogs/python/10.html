In any general programming language there is a concept about Control Flow, which means that he need to makes some checks and balances in the code. For example:- If first 10 lines of code in the program is producing some result and we need to use that in next part of code, but need to check certain aspects of the code, before doing that. This is where from concept of control flow arises. It can also be put as an analogy of stopping the water flow in a river or slowing it down with something external. In python programming language the syntax is quite simple and easy to write. For instance:-
if (a == 10):
   print("Number is 10")
else:
   print("Number is not 10")

This is how the execution of code is controlled or rather how's the execution procedes further down the program. If the statment if (a == 10) evaluates to true then next statement to it which is print("Number is 10") is executed. But in other scenario if if (a == 10) evaluates to false then next statement is skipped and else: part of the code is executed. In a sense if statement is controlling which line of code should be executed and which line should be skipped. That's why it's called Control Flow.
